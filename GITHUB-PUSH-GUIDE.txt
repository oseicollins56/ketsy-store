PART 3 — CONNECT YOUR GODADDY DOMAIN TO GITHUB PAGES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP A — Create a CNAME file in your project folder
  1. In your KETSY STORE folder, create a new text file named:  CNAME
     (exactly that — no .txt extension)
  2. Open it and type ONLY your domain name, for example:
       ketsystore.com
     (no "http://" or "www", just the bare domain)
  3. Save the file, then push it:
       git add CNAME
       git commit -m "Add custom domain CNAME"
       git push origin main

STEP B — Set custom domain in GitHub Pages
  1. Go to https://github.com/oseicollins56/ketsy-store
  2. Settings → Pages
  3. Under "Custom domain", type your GoDaddy domain (e.g. ketsystore.com)
  4. Click Save
  5. Tick "Enforce HTTPS" — GitHub provides a FREE SSL certificate

STEP C — Add DNS records in GoDaddy
  1. Log in at https://www.godaddy.com
  2. My Products → find your domain → click "DNS" or "Manage DNS"
  3. Add these records:

  ┌──────────────────────────────────────────────────────────────────┐
  │  TYPE    NAME    VALUE                              TTL           │
  ├──────────────────────────────────────────────────────────────────┤
  │  A       @       185.199.108.153                   1 Hour        │
  │  A       @       185.199.109.153                   1 Hour        │
  │  A       @       185.199.110.153                   1 Hour        │
  │  A       @       185.199.111.153                   1 Hour        │
  │  CNAME   www     oseicollins56.github.io.           1 Hour        │
  └──────────────────────────────────────────────────────────────────┘

  HOW TO ADD IN GODADDY:
  • Click "Add New Record" for each one above
  • A record: Type=A, Name=@, Value=185.199.108.153, TTL=1 Hour → Save
  • Repeat for the other 3 A records
  • CNAME: Type=CNAME, Name=www, Value=oseicollins56.github.io. → Save
  • ⚠️ Delete any old A records for @ that point to different IPs

STEP D — Wait for DNS propagation
  This takes 15 minutes to 48 hours. Check progress at:
  → https://dnschecker.org (enter your domain name)
  → Look for green checkmarks against the 4 IP addresses above

STEP E — Verify it works
  Visit your domain in a browser. You should see the Ketsy Store homepage
  loading with a padlock (HTTPS) in the address bar.

PART 4 — AFTER GOING LIVE: PAYSTACK SETTINGS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Your Paystack key is already configured in js/paystack.js.
After your domain is live, do the following:

  1. Go to https://dashboard.paystack.com → Settings → API Keys & Webhooks
  2. Under "Allowed Domains" (or "Whitelisted IPs"), add your domain:
       https://ketsystore.com  (or whatever your domain is)
  3. Confirm you are using the LIVE key starting with:  pk_live_...
  4. Test a small real payment of GHS 1 to verify everything works
  5. Check your Paystack dashboard to see the payment come through
